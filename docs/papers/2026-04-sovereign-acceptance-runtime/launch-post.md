# Launch Post

## Long-form version

AI governance cannot depend only on model providers, dashboards, or policy language.

As capable models become cheap, local, and forkable, the real control problem moves to the execution boundary.

I’m publishing a working paper called **The Sovereign Acceptance Runtime**.

It argues for a missing layer:
- AI may propose
- actions must be receipted
- contradictions must stay visible
- projection is not truth
- only humans accept canonical state

This is not a finished platform claim.
It is a doctrine paper backed by bounded proofs and an explicit open-problems list.

The goal is simple:
move AI governance from rhetoric to governed execution.
