## Path 1: SCITT Message Template

*Do not send until archive reading is complete.*

**Subject:** Scope question re: SCITT claims and agentic execution receipts

```text
Hello,

I have been reading draft-ietf-scitt-architecture-22 and have a scope question I would like the group's reaction to.

The current architecture models claims over supply-chain artifacts: issuers make statements about artifacts, notaries register them, and consumers audit. My reading is that "artifact" in the current scope is primarily a built software or data artifact — the thing produced — not the execution trace that produced it.

The question: where, in the group's view, does an agentic execution receipt sit? Specifically, when an AI agent performs a bounded action that mutates system state (a file write, a database update, a promotion of a draft to an accepted state), does the group see SCITT as the natural envelope for the receipt of that action, or as explicitly out of scope because the action is a runtime event rather than a supply-chain artifact?

I ask because I am working on an execution-boundary governance layer where the receipt of an agent action has the same properties as a supply-chain claim — issuer, subject, signature, transparency registration, auditable statement — but the subject is a state transition rather than an artifact. I have not found where this sits in the current scope discussion and would value the group's read before I write further.

For context, a short working paper describing the layer is at [your repo URL]. It is a doctrine paper, not an implementation claim; the SCITT integration question is what I am trying to clarify.

Thanks for any pointers.

[Your name]
```
