# The Sovereign Acceptance Runtime

This folder contains the release packet for the working paper.

## Files

- `paper.md` — the main paper
- `appendix.md` — claim boundaries, truth classes, and open problems
- `contributor-brief.md` — concise contributor-facing brief
- `reviewer-prompt.md` — adversarial review prompt for skeptical technical readers
- `launch-post.md` — public launch copy in multiple lengths
- `diagrams/README.md` — suggested visual diagrams for the paper

## Positioning

This is a doctrine paper, not a finished-system announcement.

It publishes:
- Sovereign Acceptance as a runtime discipline
- Tri-State Truth as an honesty protocol
- receipt-driven governance as a proof grammar
- STRONG/WEAK guard separation as a bounded execution pattern

It does not claim:
- a finished platform
- reconciled subsystem boundaries
- generalized CANON throughput
- universal artifact gating
- universal receipt unification
- complete sandboxing
